# Codebase Assurance procedure coverage

| Field-guide phase | Executable procedure | Primary output | Worked-example anchor |
| --- | --- | --- | --- |
| Understand what was supposed to be built | [Reconstruct the expected product](understand-expected-product.md) | Product expectation brief and investigation plan | Moria connects reusability, GPU authority, responsiveness, and the reference game to inspectable questions |
| Check whether the project tells one consistent story | [Review the design](review-design.md) | Design consistency review and client summary | Water, diagnostics, load ownership, latency, memory |
| Get the real product running | [Establish the baseline](reproduce-path.md) | Client-readable baseline record and raw receipts | Pinned public consumer and acceptance machines |
| Ask what convincing wrong version might still pass | [Challenge the existing evidence](design-counterexamples.md) | Shared assurance investigation and challenge receipts | Privileged consumer, skipped validator, and proxy evidence |
| Follow the promise across real boundaries | [Exercise real boundaries](exercise-composition.md) | Updated assurance investigation and boundary receipts | Public consumer, state ownership, and real Bevy/native paths |
| See what the real operator sees | [Exercise failure and recovery](exercise-operations.md) | Operator timeline and recovery proof | Developer operation is bounded; production operation remains unproven |
| Make one wrong result permanently rejectable | [Install and prove the gate](install-gate.md) | Intervention and like-for-like before/after proof | Public-interface conformance gate |
| Tell the client what the evidence permits | [Make the bounded decision](make-decision.md) | Sign-off, conditions, and residual risk | Mixed result retained without blanket pass |
